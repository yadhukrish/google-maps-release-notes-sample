# Google Maps Release Notes Sample

A Technical Writing portfolio sample demonstrating consumer product release note writing for the Google Maps mobile app (Android and iOS).

---

## About This Sample

This repository documents the Google Maps update cycle from November 2025 through March 2026 — the app's most significant release in over a decade, introducing Immersive Navigation, Gemini-powered conversational search, and landmark-based directions.

**Documentation type:** Consumer Product Release Notes  
**Audience:** End users and product teams tracking feature availability  
**Writing style:** Google Developer Documentation Style Guide  
**Release model:** Calendar-based (date of rollout, not version number)

> Google Maps follows a continuous delivery model. Features roll out progressively by region and platform rather than in discrete versioned builds. This document is structured by release month, consistent with how Google communicates Maps updates officially.

## What This Sample Demonstrates

- **Consumer vs. developer documentation:** Release notes written for end users differ significantly from developer tool changelogs. This sample uses plain language, feature-led headings, and availability callouts rather than technical diffs or API changes.
- **Calendar-based release structure:** Reflects how consumer apps with continuous delivery are documented in practice — by rollout date rather than semantic version number.
- **Availability scoping:** Each feature clearly states the platform and region it is available in, a pattern used in production release notes by Google, Apple, and Notion.
- **Completeness under constraint:** Release notes must be concise without omitting context that users need to act on the change.
